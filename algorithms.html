Algorithm Basics

An algorithm is a set of steps that solves a problem. Good algorithms are correct, clear, and efficient. Two main ways we evaluate efficiency are time complexity and space complexity.

Why efficiency matters

A solution that works for 10 items might be too slow for 10 million items. Efficiency helps us predict performance as input size grows.

Big-O examples

  • O(1): constant time (does not depend on input size)
  • O(n): linear time (grows with input)
  • O(n^2): quadratic time (nested loops)
  • O(log n): logarithmic time (divide-and-conquer, like binary search)
