Algorithm Basics Sorting Searching

Searching

Searching means finding a specific value in a dataset. The best searching method depends on whether the data is sorted and how it is stored.

Linear Search (basic idea)

Linear search checks each item one by one until it finds the target. It works on any list, even if the list is not sorted.

Typical time complexity: O(n)

Binary Search (basic idea)

Binary search repeatedly cuts a sorted list in half to find a target. Because it eliminates half the remaining items each step, it is much faster than linear search on large sorted arrays.

Typical time complexity: O(log n)

When to Use Binary Search

Binary search only works on sorted data. It repeatedly divides the search space in half, making it much faster than linear search for large datasets.

Searching in Hash Tables

Hash tables allow extremely fast average-case searching because the hash function directly maps keys to positions.
